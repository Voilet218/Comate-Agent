# PaiCLI 项目结构与模块功能总结

> 生成日期：2025-07-21
> 基于 `src/main/java/com/paicli/` 源码分析

---

## 一、项目概览

| 字段 | 值 |
|------|-----|
| **项目名称** | PaiCLI |
| **定位** | Terminal-First Agent IDE（类 Claude Code 的智能编程 Agent CLI） |
| **语言/构建** | Java 17 + Maven（单模块 fat JAR） |
| **主类** | `com.paicli.cli.Main` |
| **当前版本** | v16.1.0 |
| **代码包数** | **23 个功能模块** |

### 核心依赖栈

| 依赖 | 用途 |
|------|------|
| Jackson 2.16 | JSON 处理 |
| OkHttp 4.12 | HTTP 客户端 |
| JLine 4.0 | 终端输入交互（行编辑、补全、高亮） |
| Jieba 1.0.2 | 中文分词 |
| SQLite 3.49 | 本地持久化（长期记忆） |
| JavaParser 3.28 | Java AST 解析（LSP 诊断 / 代码分析） |
| JGit 7.6 | Git 快照操作（Side-Git） |
| Jsoup 1.18 | HTML 解析（web_fetch 网页提取） |
| Lanterna 3.1 | TUI 全屏终端界面 |
| Logback 1.5 | 日志 |
| JUnit 5 + Mockito | 单元测试 |

### 三大运行形态（通过 `PAICLI_RENDERER` 环境变量切换）

1. **inline（默认）** — Claude Code 风格的流式行内 TUI，prompt 下方 inline 状态区、可折叠工具块、行内 diff
2. **lanterna** — 全屏窗口 TUI
3. **plain** — 纯 println 兜底

## 三、23 个功能模块详解

### 1. `cli` — CLI 入口与命令行交互

**核心类**：`Main.java`、`CliCommandParser.java`、`PaiCliCompleter.java`、`PaiCliHighlighter.java`、`PaiCliHistory.java`

**功能说明**：
- 程序入口 `main()`：初始化配置、LLM 客户端、终端、渲染器、MCP、Skill 系统
- 主循环：读取用户输入，解析斜杠命令（`/model`、`/plan`、`/hitl`、`/browser`、`/mcp` 等约 40+ 命令）
- JLine 行编辑集成：Tab 补全、语法高亮、Autosuggestion、Autopair、Ctrl+O 折叠块、Ctrl+V 贴图、ESC 清空
- 三种 Agent 模式切换：ReAct（默认）、Plan-and-Execute（`/plan`）、Multi-Agent（`/team`）
- 首屏启动信息：模型、MCP 状态、Skill 摘要、操作提示

---

### 2. `agent` — Agent 核心编排

**核心类**：`Agent.java`、`PlanExecuteAgent.java`、`AgentOrchestrator.java`、`SubAgent.java`

**功能说明**：
- **`Agent`**：ReAct（推理-行动）循环核心。维护对话历史、调用 LLM、解析工具调用、执行工具、处理结果
- **`PlanExecuteAgent`**：Plan-and-Execute 模式。先让 LLM 生成多步执行计划，用户确认或补充后逐任务执行
- **`AgentOrchestrator`**：Multi-Agent 协作模式。包含规划者（Planner）、执行者（Executor）、检查者（Reviewer）三个角色
- **`SubAgent`**：协作模式中的子 Agent，执行独立子任务
- **`AgentMessage` / `AgentRole` / `AgentBudget`**：消息通信、角色定义、Token 预算管理

---

### 3. `llm` — LLM 多模型客户端

**核心接口/类**：`LlmClient`、`LlmClientFactory`、`GLMClient`、`DeepSeekClient`、`KimiClient`、`StepClient`

**功能说明**：
- `LlmClient` 接口定义了 chat、流式 chat、模型名称、上下文窗口等契约
- 四个具体实现：GLM（智谱）、DeepSeek、Kimi（Moonshot）、Step（阶跃星辰）
- `AbstractOpenAiCompatibleClient` 基类封装 OpenAI 兼容 API 的通用逻辑
- `LlmClientFactory` 根据配置/环境变量动态创建对应客户端
- 支持流式输出、图片输入（多模态）、Prompt Caching
- `LlmTraceLogger`：LLM 调用链路追踪日志

---

### 4. `tool` — 工具系统

**核心类**：`ToolRegistry.java`、`ToolOutput.java`

**功能说明**：
- `ToolRegistry`：工具注册与调度的中枢
- 内置工具：`read_file`、`write_file`、`list_dir`、`glob_files`、`grep_code`、`execute_command`、`search_code`、`web_search`、`web_fetch`、`create_project`、`revert_turn`、`save_memory`、`load_skill`
- MCP 外部工具自动注册为 `mcp__{server}__{tool}` 格式
- 并行工具调用（最多 4 个并发）
- 策略执行：安全守卫先于 HITL 审批
- 文件写入 diff 观察者回调（用于行内 diff 展示）

---

### 5. `mcp` — Model Context Protocol（MCP）协议实现

**核心类**：`McpServerManager.java`、`McpServer.java`、`McpClient.java`

**子包**：
- `config/`：MCP 配置加载（支持 `mcp.json` 和环境变量）
- `jsonrpc/`：JSON-RPC 2.0 协议实现
- `mention/`：`@server:protocol://path` 引用扩展
- `notifications/`：通知路由
- `protocol/`：协议类型定义（ToolDescriptor、ResourceDescriptor 等）
- `resources/`：MCP Resource 缓存与读取工具
- `transport/`：传输层（stdio、Streamable HTTP）

**功能说明**：
- MCP server 生命周期管理：启动、停止、重启、禁用、启用
- MCP 工具、资源（Resource）、提示（Prompt）的注册与访问
- 启动时自动创建默认 `chrome-devtools` MCP 配置
- 通过 `@mention` 在 prompt 中引用 MCP resource

---

### 6. `browser` — 浏览器集成

**核心类**：`BrowserSession.java`、`BrowserConnectivityCheck.java`、`BrowserGuard.java`

**功能说明**：
- 两种模式：`SHARED`（复用本机 Chrome 登录态）和 `ISOLATED`（临时无状态）
- `BrowserConnectivityCheck`：探活 CDP 端口（默认 9222）
- `BrowserConnector`：`/browser connect` / `disconnect` / `status` / `tabs` 命令处理
- `BrowserGuard`：敏感页面检测（登录页、银行页面等），配合 HITL 审批
- `SensitivePagePolicy`：敏感页面匹配规则
- `BrowserAuditMetadata`：浏览器操作的审计元数据

---

### 7. `memory` — 记忆系统

**核心类**：`MemoryManager.java`、`ConversationMemory.java`、`LongTermMemory.java`

**功能说明**：
- 双层记忆架构：
  - **短期记忆**（`ConversationMemory`）：当前会话的对话历史，受 Token 预算控制
  - **长期记忆**（`LongTermMemory`）：SQLite 持久化，支持项目作用域和全局作用域
- `MemoryManager` 门面类：统一存取接口，自动注入相关记忆到 system prompt
- `ContextCompressor`：上下文占用率 ≥90% 时触发 LLM 摘要压缩
- `ConversationHistoryCompactor`：对话历史压缩
- `MemoryRetriever`：短期+长期混合检索
- `TokenBudget`：Token 预算计算
- `ExplicitMemoryHints`：主动记忆提示注入

---

### 8. `plan` — 计划执行引擎

**核心类**：`Planner.java`、`ExecutionPlan.java`、`Task.java`

**功能说明**：
- `Planner`：使用 LLM 将复杂任务分解为多步执行计划
- `ExecutionPlan`：计划模型，支持 `summarize()` 和 `visualize()` 两种展示方式
- `Task`：单个任务节点（类型、名称、目标、依赖关系）
- Plan-and-Execute 模式下由 `PlanExecuteAgent` 使用

---

### 9. `hitl` — Human-in-the-Loop 人工审批

**核心接口/类**：`HitlHandler.java`、`TerminalHitlHandler.java`、`RendererHitlHandler.java`、`SwitchableHitlHandler.java`

**功能说明**：
- `HitlHandler` 接口：同步阻塞的人工审批交互契约
- 三个实现：
  - `TerminalHitlHandler`：传统终端单字符输入（y/n/a/d）
  - `RendererHitlHandler`：与渲染器集成的审批（inline/Lanterna）
  - `SwitchableHitlHandler`：可开关代理，支持 `on`/`off` 切换
- `HitlToolRegistry`：HITL 感知的工具注册表，触发审批流程
- `ApprovalPolicy`：定义哪些工具是危险的（write_file、execute_command、create_project、revert_turn + 所有 mcp__ 前缀工具）
- `ApprovalRequest / ApprovalResult`：请求与结果模型

---

### 10. `render` — 渲染引擎

**核心接口/类**：`Renderer.java`、`RendererFactory.java`、`InlineRenderer.java`、`PlainRenderer.java`

**功能说明**：
- `Renderer` 接口：流式输出、工具块、HITL、状态栏、行内 diff、palette 的抽象
- `InlineRenderer`（默认）：Claude Code 风格的行内流式渲染
  - `BottomStatusBar`：底部状态栏（模型、MCP、Skill、HITL 状态）
  - `BlockRegistry`：可折叠工具调用块，Ctrl+O 切换展开/收起
  - `InlineActivityDisplay`：活动指示器
  - `Transcript`：本轮对话回放支持
- `PlainRenderer`：纯文本 `println` 兜底
- `StatusInfo`：状态栏信息模型

---

### 11. `tui` — TUI 全屏终端界面

**核心类**：`TuiBootstrap.java`、`LanternaRenderer.java`、`LanternaWindow.java`

**子包**：
- `config/`：配置管理面板
- `highlight/`：代码语法高亮
- `history/`：对话历史可视化
- `hitl/`：HITL 审批 UI
- `pane/`：各种面板组件

**功能说明**：
- `TuiBootstrap`：检测是否启用 TUI（`PAICLI_RENDERER=lanterna`），失败时降级到 CLI
- `LanternaRenderer`：Lanterna 3.1 全屏渲染器实现
- `LanternaWindow`：主窗口（文件树、对话区、状态栏、输入区布局）
- 向后兼容：旧 `PAICLI_TUI=true` 环境变量仍可触发 TUI

---

### 12. `rag` — 代码检索增强生成

**核心类**：`CodeIndex.java`、`CodeRetriever.java`、`VectorStore.java`

**功能说明**：
- `CodeIndex`：代码库索引管道 — 遍历文件 → 分块 → 向量化 → 存储
- `CodeChunker`：将代码文件按 AST 结构分块（类、方法、字段级别）
- `CodeAnalyzer`：代码结构分析（类关系）
- `CodeRetriever`：混合检索（向量相似度 + 关键词 BM25）
- `VectorStore`：向量持久化存储
- `EmbeddingClient`：调用 LLM embedding API 生成向量
- `SearchResultFormatter`：检索结果格式化输出
- `CodeRelation`：类关系图谱数据模型
- 用户命令：`/index` 触发索引，`/search` 语义检索，`/graph` 查询类关系

---

### 13. `web` — 网络搜索与网页抓取

**核心接口/类**：`SearchProvider.java`、`WebFetcher.java`、`HtmlExtractor.java`

**功能说明**：
- `SearchProvider` 接口：搜索引擎抽象
- 三个实现：
  - `SerpApiSearchProvider`：商业聚合 API（需 API Key）
  - `SearxngSearchProvider`：开源元搜索引擎（自托管，免费）
  - `ZhipuSearchProvider`：智谱搜索
- `SearchProviderFactory`：自动选择可用的 provider
- `WebFetcher`：HTTP 网页抓取（基于 OkHttp）
- `HtmlExtractor`：HTML 转 Markdown（基于 Jsoup）
- `NetworkPolicy`：网络访问策略控制
- 对应工具：`web_search`、`web_fetch`

---

### 14. `prompt` — 提示词管理

**核心类**：`PromptAssembler.java`、`PromptRepository.java`、`PromptContext.java`

**功能说明**：
- `PromptAssembler`：按模式（react/plan/team）组装完整 system prompt
- `PromptRepository`：从 classpath 加载 `.md` 模板文件
- `PromptContext`：组装上下文（记忆上下文、MCP 资源索引、Skill 索引、审批模式等）
- `PromptMode`：模式选择枚举
- 模板分层：base.md → personalities/ → mode-specific → approvals/ → memory → skills → context-management → handoff

---

### 15. `policy` — 安全策略

**核心类**：`PathGuard.java`、`CommandGuard.java`、`AuditLog.java`

**功能说明**：
- `PathGuard`：路径围栏 — 所有文件操作强制限定在项目根内，防止 `../../etc/passwd` 越界
- `CommandGuard`：命令黑名单 — 禁止 `sudo`、`rm -rf` 全盘、`mkfs`、`dd of=/dev`、fork bomb 等
- `AuditLog`：危险工具审计日志 — 记录每次危险操作的时间、工具、耗时、审批人和原因
- `PolicyException`：策略违例异常

---

### 16. `snapshot` — Side-Git 快照系统

**核心类**：`SnapshotService.java`、`SideGitManager.java`

**功能说明**：
- `SnapshotService`：快照生命周期管理，`runTurn()` 自动包裹 pre-turn / post-turn 快照
- `SideGitManager`：基于 JGit 的 side-git 仓库操作（不干扰主 git）
- `TurnSnapshot`：快照数据模型（commitId、phase、turnId、timestamp）
- `SnapshotPhase`：阶段枚举（pre-turn / post-turn）
- `SnapshotConfig`：开关配置
- `RestoreResult`：恢复操作结果
- 用户命令：`/snapshot`、`/snapshot status`、`/snapshot clean`、`/restore <N>`

---

### 17. `runtime` — 运行时支持

**核心类**：`CancellationContext.java`、`CancellationToken.java`

**子包**：
- `api/`：Runtime HTTP API 服务器
  - `RuntimeApiServer`：HTTP API（事件推送、线程管理），支持 `serve --http` 启动
  - `RuntimeThreadStore`：线程存储
- `task/`：后台任务管理
  - `DurableTaskManager`：持久化后台任务管理
  - `TaskCommandFormatter`：`/task` 命令格式化

**功能说明**：
- 取消机制：`CancellationContext`（ThreadLocal）+ `CancellationToken`，任务运行中按 ESC 取消
- HTTP API 服务器：允许外部系统通过 REST API 调用 PaiCLI 的能力
- 后台任务：提交、查看、取消后台运行的任务

---

### 18. `skill` — 技能（Skill）系统

**核心类**：`SkillRegistry.java`、`SkillBuiltinExtractor.java`、`SkillContextBuffer.java`

**功能说明**：
- 三层扫描目录（后者覆盖前者）：
  1. **builtin**：jar 内置，解压到缓存目录
  2. **user**：`~/.paicli/skills/<name>/SKILL.md`
  3. **project**：`<project>/.paicli/skills/<name>/SKILL.md`
- `SkillRegistry`：加载、启用/禁用、查询 skill
- `SkillContextBuffer`：已启用 skill 的上下文注入
- `SkillBuiltinExtractor`：从 classpath 提取内置 skill
- `SkillFrontmatterParser`：YAML frontmatter 解析
- `SkillIndexFormatter`：skill 列表格式化
- `SkillStateStore`：JSON 持久化启用/禁用状态
- 用户命令：`/skill list`、`/skill show`、`/skill on/off`、`/skill reload`

---

### 19. `lsp` — LSP 诊断集成

**核心类**：`LspManager.java`、`LspDiagnostic.java`、`LspDiagnosticFormatter.java`

**功能说明**：
- 基于 JavaParser 的静态代码分析
- 解析 Java 文件中的编译问题和语法错误
- `LspManager`：管理诊断收集，按文件组织
- `LspDiagnosticFormatter`：格式化为 LLM 友好的诊断报告
- `LspSeverity`：严重级别（ERROR、WARNING、INFO）
- 诊断结果注入到 system prompt，帮助 LLM 生成更准确的代码

---

### 20. `config` — 配置管理

**核心类**：`PaiCliConfig.java`

**功能说明**：
- 加载/保存配置文件（`~/.paicli/config.json`）
- Provider 配置管理（API Key、模型名、端点）
- 默认 Provider 持久化
- 支持环境变量覆盖

---

### 21. `context` — 上下文分析

**核心类**：`ContextProfile.java`、`TokenUsageFormatter.java`

**功能说明**：
- `ContextProfile`：上下文策略配置，所有参数基于 `maxContextWindow` 计算
  - 压缩触发阈值：占用率 ≥ 90%
  - 短期记忆预算：window × 0.45
  - 记忆注入 Token 上限：window × 0.005，封顶 5000
  - MCP Resource 索引：window ≥ 32k 时才注入
- `TokenUsageFormatter`：Token 用量格式化输出

---

### 22. `image` — 图片处理

**核心类**：`ClipboardImage.java`、`ImageProcessor.java`、`ImageReferenceParser.java`

**功能说明**：
- `ClipboardImage`：从系统剪贴板抓取图片，保存到缓存目录
- `ImageProcessor`：图片处理（base64 编码、格式转换）
- `ImageReferenceParser`：解析 prompt 中的图片引用
- Ctrl+V 绑定：将剪贴板图片转为图片引用注入输入行

---

### 23. `util` — 通用工具

**核心类**：`AnsiStyle.java`、`JiebaSegmenterFactory.java`、`TerminalMarkdownRenderer.java`

**功能说明**：
- `AnsiStyle`：ANSI 控制序列封装（颜色、样式、加粗、灰色等）
- `JiebaSegmenterFactory`：Jieba 中文分词器工厂
- `TerminalMarkdownRenderer`：终端 Markdown 渲染（标题、列表、代码块、表格等）

## 五、设计特点

1. **单模块 + 包隔离**：没有多模块架构，通过 Java package 做逻辑隔离，保持构建简单
2. **接口驱动**：`LlmClient`、`Renderer`、`SearchProvider`、`HitlHandler` 等关键组件均面向接口编程
3. **工厂模式**：`LlmClientFactory`、`RendererFactory`、`SearchProviderFactory` 统一创建逻辑
4. **三层渲染**：inline（默认流式）/ lanterna（全屏 TUI）/ plain（纯文本兜底）
5. **三层 Skill**：builtin → user → project，逐层覆盖
6. **安全优先**：PathGuard + CommandGuard + AuditLog + HITL 审批四层防护
7. **快照恢复**：Side-Git 机制支持 turn 级撤销
